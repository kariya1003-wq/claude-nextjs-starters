# PRD: 노션 기반 견적서 웹뷰어

> 작성일: 2026-04-02
> 상태: MVP 기준

---

## 1. 프로젝트 핵심

- **목적**: 노션 DB에 입력된 견적서 데이터를 전용 URL로 클라이언트에게 즉시 공유하고, PDF로 다운로드할 수 있게 한다.
- **타겟 사용자**: 견적서를 자주 발행하는 1인 프리랜서 또는 소규모 사업자(공급자)와 해당 견적서를 수신하는 클라이언트.

---

## 2. 사용자 여정

### 전체 플로우 다이어그램

```
[공급자]
노션 DB에 견적서 입력
        │
        ▼
견적서 목록 페이지 접속
        │
        ├─ 견적서 선택 ──▶ 견적서 상세 페이지 (공급자용 미리보기)
        │                        │
        │                        └─ 고유 URL 복사 → 클라이언트에게 전달
        │
[클라이언트]
        │
        ▼
고유 URL 접속
        │
        ├─ [공개 여부 = true, 상태 = 발송|확정] ──▶ 견적서 뷰어 페이지
        │                                                   │
        │                                                   └─ PDF 다운로드
        │
        └─ [공개 여부 = false 또는 상태 = 만료] ──▶ 접근 제한 안내 페이지
```

### 페이지 간 전환 조건

| 출발 페이지          | 도착 페이지           | 전환 조건                               |
| -------------------- | --------------------- | --------------------------------------- |
| 견적서 목록 페이지   | 견적서 상세 페이지    | 목록에서 특정 견적서 클릭               |
| 견적서 상세 페이지   | 견적서 목록 페이지    | 뒤로가기 또는 목록 버튼 클릭            |
| 고유 URL (외부 링크) | 견적서 뷰어 페이지    | 공개 여부 = true, 상태가 발송 또는 확정 |
| 고유 URL (외부 링크) | 접근 제한 안내 페이지 | 공개 여부 = false 또는 상태 = 만료      |

### 사용자 선택 분기점

- **공급자**: 견적서 목록에서 특정 견적서를 선택하여 상세 확인 및 URL 복사
- **클라이언트**: 수신한 URL로 접속 시 → 유효한 견적서이면 뷰어, 만료 또는 비공개이면 접근 제한 페이지로 자동 분기

## 4. 메뉴 구조

### 헤더 메뉴 (공급자용 내비게이션)

```
[로고 / 서비스명]          [견적서 목록 → F001]
```

### 견적서 상세 페이지 내 액션

```
[← 목록으로]    [URL 복사 → F006]    [상태 배지 → F007]
```

### 견적서 뷰어 페이지 내 액션 (클라이언트용)

```
[PDF 다운로드 → F004]    [상태 배지 → F007]
```

### 접근 제한 안내 페이지

```
[홈으로 돌아가기] (견적서 목록으로 이동 불가 — 클라이언트는 목록 접근 불가)
```

> 참고: 클라이언트는 별도 로그인 없이 고유 URL로만 접근하며, 견적서 목록 페이지에 대한 내비게이션 메뉴가 노출되지 않는다.

## 5. 페이지별 상세 기능

### 5-1. 견적서 목록 페이지

- **역할**: 공급자가 노션 DB에 등록된 모든 견적서를 한눈에 파악하고, 특정 견적서로 이동하는 허브 역할
- **사용자 행동**: 견적서 목록을 확인하고, 관리하고 싶은 견적서를 클릭하여 상세 페이지로 이동
- **진입 조건**: 서비스 루트 경로에 직접 접속하거나, 헤더의 "견적서 목록" 메뉴 클릭
- **기능 목록**:
  - Notion API 호출을 통해 전체 견적서 목록을 서버에서 조회하여 렌더링
  - 각 견적서 행에 견적번호, 클라이언트명, 발행일, 상태 배지 표시
  - 각 행 클릭 시 해당 견적서 상세 페이지로 이동
  - 모바일에서 테이블이 카드 형태로 전환되는 반응형 레이아웃 적용
- **구현 기능 ID**: F001, F007, F009

---

### 5-2. 견적서 상세 페이지

- **역할**: 공급자가 특정 견적서의 전체 내용을 미리 확인하고, 클라이언트에게 전달할 공유 URL을 생성하는 페이지
- **사용자 행동**: 견적서 내용 전체를 확인하고, "URL 복사" 버튼으로 클라이언트 공유 링크를 클립보드에 복사
- **진입 조건**: 견적서 목록 페이지에서 특정 견적서 행 클릭
- **기능 목록**:
  - 선택한 견적서의 모든 필드(공급자 정보, 클라이언트명, 품목 테이블, 합계, 메모 등)를 렌더링
  - 견적서 상태 배지(초안/발송/확정/만료) 표시
  - 견적번호 기반 고유 공유 URL 생성 및 클립보드 복사 버튼 제공
  - 목록으로 돌아가는 뒤로가기 버튼 제공
  - 모바일 반응형 레이아웃 적용
- **구현 기능 ID**: F002, F006, F007, F009

---

### 5-3. 견적서 뷰어 페이지

- **역할**: 클라이언트가 공유 URL로 접속하여 견적서를 확인하고 PDF로 저장하는 최종 소비 페이지
- **사용자 행동**: 전달받은 URL로 접속하여 견적서를 읽고, 필요시 PDF 다운로드 버튼을 클릭하여 파일 저장
- **진입 조건**: 공급자로부터 전달받은 고유 URL에 직접 접속 (공개 여부 = true, 상태가 발송 또는 확정인 경우)
- **기능 목록**:
  - 견적서 번호 기준으로 Notion API에서 데이터 조회 후 견적서 양식으로 렌더링
  - 공급자 정보, 클라이언트명, 품목 테이블(품목명, 수량, 단가, 합계), 총액, 유효기간, 메모 표시
  - 견적서 상태 배지 표시 (발송 또는 확정 상태만 접근 가능)
  - "PDF 다운로드" 버튼 클릭 시 현재 견적서를 PDF로 변환하여 파일 다운로드
  - OG 메타태그 동적 설정 (견적번호, 클라이언트명 포함)
  - 모바일 반응형 레이아웃 적용
- **구현 기능 ID**: F002, F003, F004, F007, F008, F009

---

### 5-4. 접근 제한 안내 페이지

- **역할**: 유효하지 않은 견적서 URL로 접근한 클라이언트에게 명확한 안내 메시지를 전달하는 페이지
- **사용자 행동**: 안내 메시지를 읽고 공급자에게 문의하거나, 페이지를 닫음
- **진입 조건**: 공유 URL로 접속 시 공개 여부가 false이거나 상태가 만료인 경우 자동 리디렉션
- **기능 목록**:
  - "견적서에 접근할 수 없습니다" 안내 메시지 표시
  - 접근 불가 사유 안내 (만료 또는 비공개 처리됨)
  - 공급자 문의를 유도하는 안내 문구 표시
- **구현 기능 ID**: F005

## 6. 데이터 모델

노션 DB를 직접 데이터 저장소로 사용하므로, 별도 DB 테이블은 없음. 아래는 Notion DB 필드 기준 데이터 모델.

### Quote (견적서)

| 필드명       | 설명                                            |
| ------------ | ----------------------------------------------- |
| quoteNumber  | 고유 식별자 (예: Q-2024-001), Notion Title 필드 |
| clientName   | 수신 업체명                                     |
| issueDate    | 견적서 발행 날짜                                |
| expiryDate   | 견적 유효 만료일                                |
| items        | 품목 리스트 (JSON: 품목명, 수량, 단가, 합계)    |
| supplierInfo | 공급자 회사명, 담당자, 연락처                   |
| memo         | 특이사항 및 추가 안내                           |
| status       | 초안 / 발송 / 확정 / 만료                       |
| isPublic     | 클라이언트 접근 허용 여부 (boolean)             |

### PDF 생성

| 기술          | 선택 이유                                                                                                                                                                                                                                                                        |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **puppeteer** | 서버 사이드에서 실제 브라우저 엔진으로 렌더링 후 PDF 변환하므로, 웹에서 보이는 스타일이 PDF에 그대로 적용됨. Tailwind CSS 스타일과의 호환성이 높고, 복잡한 레이아웃도 정확히 출력 가능. `react-pdf`는 별도 PDF 전용 컴포넌트를 이중으로 작성해야 하는 유지보수 부담이 있어 제외. |

> **참고**: puppeteer는 서버 환경(Next.js Route Handler 또는 Server Action)에서 실행되며, 클라이언트 번들 사이즈에 영향을 주지 않음.
